'In order to get good data, you will need to calibrate the sensor.  In the LoggerPro window – first open the file “Water Column.cmbl”.'
It looks like the students should be able to get good results without going through this calibration procedure. I believe I will be able to provide a *.cmbl file with the calibration information already entered and the students will only need to hit the "zero" tool on the toolbar within loggerpro (much like we zero our electronic balances). This should streamline this part of the lab and avoid the need to use the water column to calibrate the pressure sensor which would then be used to investigate static pressure in a water column. Someone had pointed out that is circular. The sensor manufacturer actually provides enough information about the sensor for the calibration.
See ../waterColumn/StaticWaterPressure.html#calibration_sensor

In order to get this to work I purchased independent 5 volt DC supplies for each sensor and each will be permanently attached to a sensor.
I think they need to be permanently attached so that
  1. They will not get mixed up with the 9 Vdc supplies we will be using with the 5 V breadboard regulator boards.
  2. If it turns out there is some small variation between one supply and the next significant enough to affect the calibration that I will be able to put a calibration sticker on each sensor box and the correct configuration file for that specific box can be selected (or the corrected equation can be entered).
